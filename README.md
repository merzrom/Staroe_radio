# Staroe_radio

Десктопный аудиоплеер для потокового прослушивания и скачивания записей проекта "Старое радио"
с русскоязычных аудиоархивов (staroeradio.ru, lektorium.su, reportage.su, svidetel.su, theatrologia.su,).

Написан на Python (tkinter + ttkbootstrap), воспроизведение — через libVLC.

## Возможности

- Поиск по локальным текстовым базам записей нескольких источников
- Потоковое воспроизведение MP3 по HTTP через VLC (libvlc)
- Запоминание позиции воспроизведения — продолжение с того же места после перезапуска,
  даже если список результатов поиска изменился
- Загрузка программы передач (расписания) — в том числе автоматически при первом запуске
- Описание трека с подгрузкой изображений со страницы источника
- Скачивание отдельных треков и целых результатов поиска в MP3 с ID3-тегами
- Сохранение плейлистов в формате M3U (один трек или все результаты поиска)
- Выбор папки для сохранения через стандартный диалог, с запоминанием последней папки
  (создаётся подпапка `Staroe_radio_downloads`, для пакетных загрузок — вложенная папка по запросу)
- Избранное (`favorites.txt`) и история прослушивания (папка `History`)
- Контекстные меню (копировать/вставить, скачать, сохранить в плейлист)
- Тёмная тема интерфейса (ttkbootstrap, `darkly`)
- Настраиваемые цвета через `colors_config.json`

## Установка

### 1. Зависимости Python

```bash
pip install -r requirements.txt
```

### 2. VLC

Для воспроизведения требуется VLC media player (точнее, библиотеки `libvlc.dll`
и `libvlccore.dll` и папка `plugins` с кодеками).

- Установите VLC: https://www.videolan.org/vlc/
- Либо скопируйте `libvlc.dll`, `libvlccore.dll` и папку `plugins` из установки VLC
  в директорию со скриптом (актуально для сборки в `.exe`)

## Запуск

```bash
python SRPlayer.py
```

При первом запуске (если ещё нет `player_state.json`) автоматически загружается
программа передач на сегодня.

## Базы данных источников

Текстовые файлы в репозитории — локальные базы записей по источникам:

| Файл | Источник |
|---|---|
| `staroeradio.txt` | staroeradio.ru |
| `lektorium.txt` | lektorium.su |
| `svidetel.txt` | svidetel.su |
| `reportage.txt` | reportage.su |
| `theatrologia.txt` | тtheatrologia.su |

Формат строки: `ID<TAB>Название записи`.

## Сборка в .exe (Windows)

Используется PyInstaller с готовым `.spec`-файлом:

```bash
pyinstaller SRPlayer.spec
```

Перед сборкой убедитесь, что `libvlc.dll`, `libvlccore.dll` и папка `plugins`
находятся рядом со скриптом — они должны быть включены в итоговую сборку.

## Структура пользовательских файлов

При работе программа создаёт/использует:

- `player_state.json` — сохранённое состояние (позиция воспроизведения, размеры окон,
  последняя папка сохранения и т.д.)
- `favorites.txt` — избранные записи
- `History/` — история прослушивания
- `colors_config.json` — настройки цветовой схемы интерфейса

Эти файлы специфичны для пользователя и не входят в репозиторий (см. `.gitignore`).
